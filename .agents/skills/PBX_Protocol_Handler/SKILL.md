# ทักษะ PBX Protocol Handler

ทักษะนี้มีไว้สำหรับช่วยให้ Agent แปลงคำสั่งระดับสูง (เช่น สั่ง Check-in หรือสั่ง Check-out) ไปเป็นคำสั่ง ASCII Protocol ที่สามารถสื่อสารกับตู้ Phonik PBX (ผ่านพอร์ต LAN ของPBX หรือ TCP) และสั่งงานบอร์ดรีเลย์ ECS-103R V.5 ในห้องพักได้

## ⚙️ ข้อกำหนดทางโปรโตคอล (Phonik ASCII Protocol Spec)

1. **โครงสร้างของคำสั่ง (Command)**:
   - นำหน้าด้วยสัญลักษณ์สองจุด: `..`
   - ลงท้ายคำสั่งด้วย Carriage Return และ Line Feed: `\r\n` (CR+LF)
   
2. **โครงสร้างของการตอบกลับ (Response)**:
   - นำหน้าด้วยสัญลักษณ์เครื่องหมายเท่ากับสองตัว: `==`
   - ลงท้ายด้วย `\r\n`
   - หากคำสั่งผิดพลาดหรือไม่เป็นที่ยอมรับ จะตอบกลับด้วย: `==NACK\r\n`

## 📋 รายการคำสั่งสำคัญ (Core Command Reference)

| การทำงาน (Action) | รูปแบบคำสั่ง (Command Format) | ตัวอย่างคำสั่ง | คำอธิบาย |
| :--- | :--- | :--- | :--- |
| **Check-in / เปิดไฟ** | `..ROOM{room_id}=1\r\n` | `..ROOM0101=1\r\n` | สั่งเปิดระบบไฟในห้อง (จ่ายไฟ 220V เข้าวงจร) |
| **Check-out / ตัดไฟ** | `..ROOM{room_id}=0\r\n` | `..ROOM0101=0\r\n` | สั่งปิดระบบไฟในห้องทันที |
| **อ่านสถานะไฟฟ้าห้อง** | `..ROOM{room_id}=\r\n` | `..ROOM0101=\r\n` | อ่านสถานะปัจจุบัน (0=OFF, 1=ON, 2=MAINT, 3=OOO) |
| **บันทึกชื่อผู้เข้าพัก** | `..NAME{room_id}={name}\r\n` | `..NAME0101=John Doe\r\n` | บันทึกชื่อแขกในตู้สาขา (ความยาวไม่เกิน 16 ตัวอักษร) |
| **อ่านชื่อผู้เข้าพัก** | `..NAME{room_id}=\r\n` | `..NAME0101=\r\n` | ดึงชื่อผู้เข้าพักปัจจุบันในห้องนั้นออกมา |
| **ตรวจเวอร์ชันตู้สาขา** | `..VERS=\r\n` | `..VERS=\r\n` | ตรวจสอบรุ่นเฟิร์มแวร์ (ใช้เป็น Heartbeat/Ping ได้) |
| **ตัดการเชื่อมต่อ** | `..STOP\r\n` | `..STOP\r\n` | ยกเลิกการเชื่อมต่อกับตู้สาขาอย่างสง่างาม |

## 🛡️ กฎการทำงานของ Agent เมื่อใช้ทักษะนี้

1. **Normalizing Room ID**: ทุกครั้งที่ได้รับ Room ID (เช่น 101, "203") จะต้องแปลงให้อยู่ในรูปแบบ 4 หลักเสมอ โดยการเติม 0 ด้านหน้า (Zero-padding) เช่น `101` -> `0101`, `203` -> `0203`
2. **Name Sanitization**: สำหรับคำสั่งบันทึกชื่อ แขกต้องถูกตัดคำไม่ให้เกิน 16 ตัวอักษร (ASCII-safe) และทำการตัดช่องว่างหน้าหลัง (Trim) เสมอ
3. **Safety First**: ก่อนเขียนโค้ดเพื่อส่งชุดคำสั่งเหล่านี้ ให้มองหาหรือสร้าง Unit Test ในส่วนของ Protocol Engine ก่อนเสมอเพื่อป้องกันความผิดพลาดทางโครงสร้างคำสั่ง
