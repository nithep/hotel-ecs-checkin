# 📖 คู่มือการใช้งานระบบ Smart Hotel Self Check-in (ฉบับผู้ใช้งานจริง)

**เอกสารสำหรับ:** พนักงานต้อนรับ (Receptionist) และคู่มือสำหรับแขก (Guest Guide)  
**เวอร์ชันของระบบ:** 1.5 (รองรับระบบป้องกันจอดำและป้องกันคิวตู้สาขาค้าง)

---

## 🏨 1. ภาพรวมระบบ (System Overview)
ระบบ Hotel ECS (Electronic Control System) คือ Web Application ที่ทำงานเชื่อมต่อกับตู้สาขาโทรศัพท์ (Phonik PBX) เพื่อควบคุมการจ่ายไฟ 220V ในห้องพักโดยอัตโนมัติ โดยตัดกระบวนการที่พนักงานต้องกดเปิดห้องผ่านโปรแกรมเก่า (Room Manager) ออกไป เปลี่ยนเป็นการให้แขกสแกนเช็คอินด้วยตนเอง (Self Check-in) 

---

## 👩‍💻 2. สำหรับพนักงานต้อนรับ (Hotel Staff / Admin)

### 2.1 การเข้าสู่ระบบ Dashboard
1. เปิดเบราว์เซอร์ (Google Chrome หรือ Safari)
2. พิมพ์ที่อยู่ IP ของเครื่อง Raspberry Pi (เช่น `http://192.168.1.109:5173` หรือโดเมนที่ตั้งไว้)
3. ล็อกอินเข้าสู่ระบบด้วยรหัสผ่านของพนักงาน

### 2.2 การมอนิเตอร์สถานะห้องพัก (Room Monitoring)
- ในหน้า **คอนโซล (Console)** พนักงานสามารถเห็นสถานะห้องพักทุกห้องได้แบบ **Real-time**
- **สัญลักษณ์สถานะ:**
  - 🟢 **เปิดใช้งาน (Occupied):** ระบบไฟถูกจ่ายเข้าห้อง แขกสามารถเสียบคีย์การ์ดเพื่อเปิดแอร์/ไฟได้
  - ⚪ **ว่าง (Vacant):** ห้องถูกตัดไฟสมบูรณ์
- ระบบมีการดึงสถานะ (Heartbeat) จากตู้ PBX อย่างต่อเนื่อง หากตู้สาขาขัดข้อง จะมีแจ้งเตือนสีแดงแสดงที่มุมจอ

### 2.3 การสั่งการแบบ Manual (Manual Override)
ในกรณีที่แขกไม่สะดวกสแกน QR พนักงานสามารถเปิดไฟให้แขกได้เอง:
1. ไปที่เมนู **ควบคุมห้องพัก**
2. เลือกหมายเลขห้องที่ต้องการ (เช่น 101)
3. กดปุ่ม **"เปิดไฟ (Check-in)"** หรือ **"ตัดไฟ (Check-out)"**
4. ระบบจะส่งคำสั่งเข้าตู้สาขา หากสำเร็จ ไฟในห้องแขกจะติดทันที 

### 2.4 การรับมือเมื่อระบบขัดข้องชั่วคราว
- **หน้าจอพนักงานแสดง Error สีแดง (Premium Error Boundary):**  
  หากอินเทอร์เน็ตหลุดหรือมีข้อผิดพลาดร้ายแรง หน้าจอจะแสดงหน้าต่าง "ระบบขัดข้องชั่วคราว" แบบพรีเมียม ให้พนักงานกดปุ่ม **"รีโหลดหน้าจอ"** ระบบจะซ่อมแซมตัวเองและกลับมาทำงานต่อได้โดยที่ข้อมูลไม่หาย
- **คำสั่งเปิดไฟช้า (Queue Timeout):**  
  หากสั่งเปิดไฟแล้วระบบหน่วงเกิน 5 วินาที ระบบจะยกเลิกคำสั่งนั้นเพื่อไม่ให้คิวห้องอื่นค้าง ให้พนักงานกดสั่งเปิดไฟใหม่อีกครั้ง

## 📱 3. สำหรับแขกผู้เข้าพัก (Guest Self Check-in)

พนักงานสามารถนำขั้นตอนเหล่านี้ไปอธิบายหรือทำป้ายตั้งโต๊ะ (Standee) ให้แขกได้:

### 3.1 ขั้นตอนการเช็คอินด้วยตนเอง
1. **สแกน QR Code** ที่เคาน์เตอร์ หรือหน้าห้องพัก
2. **ยินยอมสิทธิ์ (Permissions):** 
   - ระบบจะเด้งหน้าจอขออนุญาตเข้าถึงกล้อง (Allow Camera)
   - หากสแกนผ่านแอป LINE (LIFF) ระบบจะเปิดกล้องให้แนบเนียนโดยไม่ต้องติดตั้งแอปเพิ่ม
3. **การสแกนใบจอง / QR ส่วนตัว:** หันกล้องไปที่ QR Code ใบจองของแขก
4. **สำเร็จ:** หน้าจอจะขึ้นเครื่องหมายติ๊กถูกสีเขียว (Success) และระบบไฟในห้องพักจะดัง "แป๊ก" (จ่ายไฟ) ทันที
5. แขกเข้าห้องพักและเสียบคีย์การ์ดเพื่อใช้งานห้องได้ตามปกติ

### 3.2 การแก้ปัญหาสำหรับแขก (Guest Troubleshooting)
- **สแกนแล้วจอดำ:**  
  *สาเหตุ:* โทรศัพท์ของแขกไม่ได้เปิดการอนุญาตกล้อง หรือใช้งานผ่านโหมด Incognito ที่ปิดกั้นกล้อง  
  *วิธีแก้:* ระบบจะแสดงหน้าจอแจ้งเตือนให้แขกทราบชัดเจน (ไม่ใช่หน้าจอดำเปล่าๆ) พร้อมปุ่มคำแนะนำในการไปที่ `Settings > Allow Camera`
- **ต้องการเช็คเอาท์:**  
  เมื่อแขกต้องการคืนห้อง สามารถกดปุ่ม **"Check-out"** บนมือถือ ระบบจะตัดไฟห้องทันที (ไม่ว่าแขกจะลืมดึงคีย์การ์ดออกหรือไม่ก็ตาม)

## 🚨 4. สถานการณ์ฉุกเฉินและการกู้คืน (Emergency SOP)

1. **ไฟดับทั้งโรงแรม:**
   - เมื่อไฟมา Raspberry Pi จะเปิดตัวเองอัตโนมัติ 
   - สถานะของห้องที่เช็คอินไปแล้วจะถูกดึงมาจากระบบฐานข้อมูล (Database) เพื่อยิงคำสั่งฟื้นฟู (Recovery) กลับไปยังตู้ PBX พนักงานไม่ต้องทำอะไรเพิ่มเติม
2. **เชื่อมต่อตู้ PBX ไม่ได้:**
   - หากไฟสถานะมุมขวาล่างของ Dashboard ขึ้นว่า `PBX Disconnected` ให้ช่างตรวจสอบสาย LAN ระหว่างตู้สาขา (Phonik) กับเร้าเตอร์
