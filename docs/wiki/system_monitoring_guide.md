# 📊 คู่มือระบบติดตามและรายงานสถานะ Hotel ECS

โปรเจกต์นี้มีระบบตรวจสอบและรายงานสถานะ (Monitoring) ที่เชื่อมโยงกับฮาร์ดแวร์ตู้สาขา (PBX) และฐานข้อมูลโดยตรง โดยถูกออกแบบมาให้ส่งการแจ้งเตือนเข้าสู่ **Google Chat** ของทีมบริการส่วนหน้า (Front Desk / Operations) ได้ทันที 

ระบบแบ่งออกเป็น 3 ส่วนหลัก ดังนี้:

## 1. 🚨 Event-Driven Alerts (ระบบแจ้งเตือนฉุกเฉินอัตโนมัติ)
- **พฤติกรรม:** ระบบจะทำงานอยู่เบื้องหลังเงียบๆ และจะส่งการ์ดแจ้งเตือน (Alert Card) เข้า Google Chat **เฉพาะเวลาที่ตู้สาขา (PBX) ขาดการเชื่อมต่อ** (เช่น สาย LAN หลุด, ไฟดับ)
- **การเชื่อมโยง:** Node.js Backend ดักจับ Event `connection_lost` จากโมดูล PBX Connector และเรียกใช้ `GoogleNotifier.sendSystemAlert()` ทันทีโดยไม่ต้องรอ
- **วิธีใช้งาน:** ไม่ต้องทำอะไรเลย ระบบทำงานอัตโนมัติ 24 ชม.

## 2. 📅 Daily Cronjob Report (รายงานสรุปยอดตอนสิ้นวัน)
- **พฤติกรรม:** ทุกๆ คืนเวลา **23:55 น.** ระบบจะดึงข้อมูลจากฐานข้อมูล SQLite (`hotel.db`) เพื่อนับยอดผู้เข้าพักและตรวจสอบสถานะออนไลน์ของตู้ PBX จากนั้นจะจัดทำรายงานส่งเข้า Google Chat
- **การเชื่อมโยง:** ใช้ไลบรารี `node-cron` ฝังอยู่ใน Backend รันแบบ Background Process 
- **ตัวอย่างข้อมูลในรายงาน:** ยอดห้องพักทั้งหมด, จำนวนห้องที่มีแขก, จำนวนห้องว่าง, อัตราเข้าพัก (%), และสถานะ PBX

## 3. 📲 On-Demand Report (สั่งเรียกดูรายงานทันที)
คุณสามารถสั่งให้ระบบสร้างรายงานสรุป ณ เวลานั้นได้ทันทีโดยไม่ต้องรอเวลาดึก โดยการเรียก (Trigger) ผ่าน API Endpoint

**Endpoint:** `GET /api/system/report`

### วิธีการเรียกใช้งานแบบง่ายๆ (สำหรับผู้ใช้)

**วิธีที่ 1: ผ่านเว็บบราวเซอร์**
สามารถพิมพ์ URL ต่อไปนี้ลงใน Google Chrome หรือเบราว์เซอร์ใดก็ได้บนมือถือหรือคอมพิวเตอร์ที่ต่อเน็ตวงเดียวกัน:
```
http://192.168.1.94:3000/api/system/report
```
*หรือถ้าเชื่อมผ่านโดเมนภายนอก:*
```
https://hotel.nithep.com/api/system/report
```
เมื่อกด Enter คุณจะเห็นข้อความสรุป และในขณะเดียวกัน **การ์ดรายงานจะถูกส่งเข้า Google Chat ทันที**

**วิธีที่ 2: ผ่าน AppSheet (Google Workspace)**
1. ไปที่แอป AppSheet ของโรงแรม
2. สร้างปุ่ม Action ชนิด **"Data: execute an action on a set of rows"** -> **"Call a webhook"**
3. ตั้งค่า URL Webhook ไปที่ `https://hotel.nithep.com/api/system/report` (Method: GET)
4. เมื่อพนักงานกดปุ่มนี้ ระบบจะส่งรายงานเข้ากลุ่มแชททันที

**วิธีที่ 3: ผ่านคำสั่ง PowerShell (สำหรับโปรแกรมเมอร์/ช่าง)**
เปิด PowerShell แล้วรันคำสั่งบรรทัดเดียว:
```powershell
Invoke-RestMethod -Uri "http://192.168.1.94:3000/api/system/report"
```

---
*ระบบได้รับการออกแบบมาเพื่อให้ Edge Server (Raspberry Pi 4) ทำงานแจ้งเตือนด้วยตัวเองอย่างสมบูรณ์แบบ ลดภาระการเฝ้าระวังของมนุษย์ 100%*
