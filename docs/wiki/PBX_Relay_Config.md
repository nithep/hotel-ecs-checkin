# 🛠️ คู่มือการตั้งค่าตู้สาขา PBX (Phonik Config Builder)

เพื่อให้ระบบ Smart Hotel Self Check-in ใช้งานได้จริงและทำงานสอดคล้องกับซอฟต์แวร์ Node.js ของเรา จะต้องตั้งค่าพารามิเตอร์ **"P" (Program)** ในโปรแกรม **Phonik Config Builder 3.0.0** ให้ตรงกับหมายเลขห้องที่ระบุไว้ใน Frontend/Backend ดังนี้:

## 🔌 1. การแมปหมายเลขรีเลย์กับห้องพัก (P500 - Relay Number)
หมวดหมู่: `P5XX Relay & Sensor Programming > Relay`
ตั้งค่าให้ Relay ของฮาร์ดแวร์ ECS-103R ผูกกับเบอร์ห้อง (Extension) ที่ต้องการ เพื่อให้คำสั่ง `..PWER<เบอร์ห้อง>=1` ทำงานได้ถูกต้อง

| Relay No. | P500 (Relay Number / เบอร์ห้อง) | Group |
| :---: | :---: | :---: |
| 33 | **101** | 32 |
| 34 | **102** | 32 |
| 35 | **103** | 32 |
| 36 | **104** | 32 |
| 37 | **105** | 32 |
| 38 | **106** | 32 |

*(หมายเหตุ: ต้องตั้งหมายเลขห้องใน P500 ให้ตรงกับฐานข้อมูล SQLite และ Frontend `101`-`106` เป๊ะๆ)*

## ⚙️ 2. การตั้งค่าโหมดการทำงานของรีเลย์ (Relay Setting)
เพื่อให้รีเลย์จ่ายไฟค้างไว้เมื่อ Check-in และตัดไฟทันทีเมื่อ Check-out ต้องตั้งค่าดังนี้:

- **P502 (Relay Mode)** = `0` (โหมดปกติ - Normal/Continuous)
- **P503 (Hold Time)** = `00:05` (เวลาหน่วงมาตรฐานของระบบ Phonik)
- **P504 (Inverse Mode)** = `Unchecked` (ไม่กลับขั้ว)
- **P506 (ECS)** = `Checked/Assigned` (หากมีให้เลือก เพื่อระบุว่าเป็นบอร์ด ECS)

## 🗄️ 3. การกำหนด Hardware Assignment
หมวดหมู่: `Hardware Assignment`
- **P505 (Relay Slot)** ต้องถูกระบุให้ตรงกับช่อง Slot ของการ์ดรีเลย์ที่เสียบอยู่ในตู้สาขา (เช่น Slot 1, 2, 3...) เพื่อให้ตู้สาขารู้ว่าจะต้องส่งสัญญาณไปที่บอร์ดไหน

---
**✅ สถานะปัจจุบัน:** หมายเลขห้องในระบบ Frontend และ Backend ตั้งตรงกับ P500 ของตู้สาขาเรียบร้อยแล้ว ระบบพร้อมใช้งานจริง (Go-Live) ได้ทันที
