# 🌐 คู่มือการตั้งค่าโดเมน www.nithep.com เข้าสู่ตู้ควบคุม Pi 4 (ผ่าน Cloudflare Tunnel)

เพื่อที่จะทำให้โทรศัพท์มือถือของคุณ หรือลูกค้าข้างนอกสามารถสแกนเข้าใช้งานหน้าเว็บผ่านชื่อโดเมนของคุณเอง (เช่น `www.nithep.com` หรือ `hotel.nithep.com`) โดยไม่ต้องใช้เลข IP 192.168.x.x และมีความปลอดภัยสูงระดับสากล เราจะใช้เทคโนโลยี **Cloudflare Tunnel** ครับ

ข้อดีของวิธีนี้คือ **คุณไม่จำเป็นต้องขอ Public IP จากค่ายเน็ต ไม่ต้อง Forward Port ในเราเตอร์ให้เสี่ยงโดนแฮก** และระบบจะสร้างใบรับรอง SSL (https://) ให้คุณใช้งานฟรีโดยอัตโนมัติ

---

## 🛠️ ขั้นตอนที่ 1: ชี้โดเมน nithep.com เข้าสู่ Cloudflare (ฟรี)
1. สมัครสมาชิกเว็บ [Cloudflare](https://www.cloudflare.com)
2. กด **Add a Site** แล้วกรอกโดเมนของคุณ: `nithep.com`
3. เลือกแพลนแบบ **Free**
4. Cloudflare จะสแกน DNS เดิมของคุณ แล้วแสดงค่า **Nameservers** ใหม่มาให้ (เช่น `alice.ns.cloudflare.com` และ `bob.ns.cloudflare.com`)
5. ให้คุณเข้าไปที่ผู้ให้บริการจดโดเมนที่คุณซื้อไว้ (Domain Registrar เช่น GoDaddy, Namecheap หรืออื่นๆ) แล้วแก้ไข **Nameservers** ของโดเมนคุณให้ชี้มาตามค่าที่ Cloudflare ให้มา
6. รอระบบอัปเดต (ประมาณ 5-30 นาที) โดเมนของคุณก็จะมาบริหารจัดการผ่าน Cloudflare เรียบร้อย

---

## 🔑 ขั้นตอนที่ 2: สร้าง Cloudflare Tunnel
1. ในหน้าแดชบอร์ด Cloudflare เมนูด้านซ้ายกดเข้า **Zero Trust**
2. ไปที่ **Networks** > **Tunnels** แล้วกด **Create a tunnel**
3. ตั้งชื่ออุโมงค์ว่า `hotel-pi4-tunnel` แล้วกดบันทึก
4. ระบบจะแสดงคำสั่งติดตั้งสำหรับ OS ต่างๆ ให้เลือกแท็บ **Docker**
5. คัดลอกโทเค็นที่เป็นคีย์ยาวๆ หลังคำว่า `--token` เช่น:
   `eyJhIjoiM2MwOWRj...`
6. นำโทเค็นนี้ไปใส่ในไฟล์ `.env` ที่อยู่ในโฟลเดอร์ของเซิร์ฟเวอร์เราบน Pi 4:
   ```env
   CLOUDFLARE_TUNNEL_TOKEN="คีย์โทเค็นยาวๆที่ก๊อปมา"
   ```

## 🔗 ขั้นตอนที่ 3: ตั้งค่าการแมปโดเมน (Public Hostnames)
1. ในหน้าสร้างอุโมงค์ของ Cloudflare เมื่อสร้างเสร็จแล้ว ให้กดแท็บ **Public Hostname** ด้านบน
2. กด **Add a public hostname**
3. กรอกข้อมูลดังนี้:
   - **Subdomain:** `hotel` (หากต้องการให้เข้าผ่าน `hotel.nithep.com`) หรือใส่ `www` (หากต้องการให้เข้าผ่าน `www.nithep.com`)
   - **Domain:** เลือก `nithep.com`
   - **Type:** เลือก `HTTP`
   - **URL:** ใส่ `hotel-backend:3000` (หากรันผ่าน Docker Compose) หรือใส่ `localhost:3000` (หากรันบนวินโดวส์จำลอง)
4. กด **Save hostname**

เพียงเท่านี้ก็เสร็จสิ้น! ทุกคนที่พิมพ์ `https://hotel.nithep.com` หรือ `https://www.nithep.com` จากทั่วโลก จะถูกวิ่งทะลุอุโมงค์ที่ปลอดภัยของ Cloudflare ตรงเข้าสู่พอร์ต 3000 ของ Raspberry Pi 4 ในโรงแรมของคุณทันทีอย่างมหัศจรรย์!

## 📧 ขั้นตอนที่ 4: การยืนยันโดเมนกับ Google Workspace (สำหรับระบบอีเมล/แชท)
จากหน้าจอที่คุณทำการปลดล็อกฟีเจอร์พรีเมียมใน Google Workspace สำเร็จแล้ว:
1. กดปุ่มสีน้ำเงิน **"สร้างอีเมลที่กำหนดเอง"**
2. ระบบจะให้คุณใส่โดเมนที่คุณครอบครอง ให้ใส่โดเมน `nithep.com`
3. จากนั้น Google จะขอให้คุณยืนยันความเป็นเจ้าของ (Domain Verification) วิธีการยืนยันที่ง่ายที่สุดคือ:
   - เลือกวิธียืนยันด้วย **TXT Record**
   - คัดลอกค่า TXT ที่ Google ให้มา (มักจะขึ้นต้นด้วย `google-site-verification=...`)
   - กลับมาที่หน้าจัดการ DNS ของโดเมนคุณใน **Cloudflare** กด **Add Record**
   - เลือก Type: `TXT` / Name: `@` / Content: วางค่าที่ก๊อปปี้มาจาก Google
   - กด Save แล้วกลับมากดปุ่ม **"Verify"** ใน Google Workspace
4. เมื่อผ่านการยืนยันแล้ว บัญชีแอดมินองค์กรของคุณจะทำงานโดยสมบูรณ์ และสิทธิ์ในการสร้าง Webhook บน Google Chat ก็จะปลดล็อกให้ใช้งานได้ทันทีครับ!
