# 🏗️ คู่มือสรุปโครงสร้างพื้นฐานและการติดตั้ง (Infrastructure & Hosting Setup)

เอกสารฉบับนี้สรุปภาพรวมทั้งหมดของการติดตั้งระบบ **Hotel-ECS (Smart Check-in)** ตั้งแต่การเตรียมฮาร์ดแวร์ OS ไปจนถึงการเปิดให้บริการเว็บไซต์สู่โลกภายนอก (Public Hosting) โดยมีการบันทึกข้อมูลสำคัญและรหัสผ่านแยกไว้เพื่อให้ทีมงานเรียกใช้งานได้ง่ายและปลอดภัย

---

## 1. 🖥️ ฮาร์ดแวร์และระบบปฏิบัติการ (Hardware & OS)
* **อุปกรณ์:** Raspberry Pi 4 Model B
* **ระบบปฏิบัติการ:** Raspberry Pi OS Lite (64-bit) Bookworm
* **เหตุผลการเลือก:** ใช้รุ่น Lite แบบไม่มีหน้าจอ (Headless) เพื่อประหยัดทรัพยากร CPU/RAM ลดความร้อน และรองรับการทำงานแบบเซิร์ฟเวอร์เปิด 24 ชม. ได้เสถียรที่สุด

> [!IMPORTANT]
> **บัญชีผู้ใช้งานเครื่อง Raspberry Pi (SSH Login)**
> - **Username:** `pi4`
> - **Password:** `555`

## 2. 🛜 การตั้งค่าเครือข่าย (Network Setup)
ตัวบอร์ด Raspberry Pi จำเป็นต้องเชื่อมต่อเครือข่าย 2 ส่วน:
1. **Wi-Fi / อินเทอร์เน็ต (`wlan0`):** สำหรับดึงข้อมูลเวลา, อัปเดตระบบ, และส่งข้อมูลออกสู่โลกภายนอก
2. **LAN / Ethernet (`eth0`):** สำหรับเชื่อมต่อตรงเข้ากับตู้สาขา (Phonik PBX) 

> [!NOTE]
> **การล็อกหมายเลข IP (DHCP Binding / Static IP)**
> เพื่อให้ระบบมีเสถียรภาพสูงสุดและป้องกันปัญหา IP ชนกัน เราได้ทำการ Fix IP ไว้ที่ระดับเร้าเตอร์ (DHCP Binding) ซึ่งเป็นไปตามหลักวิชาการและ Best Practice ที่ดีกว่าการไป Fix ที่ตัว OS โดยตรง ข้อมูลการล็อก IP มีดังนี้:
> - **Pi4-SSID1 (Wi-Fi):** MAC `88:a2:9e:11:07:fe` ➡️ IP `192.168.1.109`
> - **Pi4-LAN1 (LAN):** MAC `88:a2:9e:11:07:fd` ➡️ IP `192.168.1.94`

> [!TIP]
> **ข้อมูล Wi-Fi (WPA-PSK)**
> - **เป้าหมายหน้างาน:** `UFI-651838`
> - **เป้าหมายตั้งค่าสำรอง:** `NT-WIFI_2.4G` (รหัสผ่าน: `9999988888`)
> 
> *การจัดการ Wi-Fi ถูกควบคุมโดยเครื่องมือ `nmcli` (NetworkManager) และสามารถสลับคลื่นผ่าน Web UI `/wifi` ของระบบได้*

## 3. ⚙️ ซอฟต์แวร์สแต็กและคอนเทนเนอร์ (Software Stack)
ระบบเบื้องหลังถูกแพ็กและจัดการผ่าน **Docker Compose** เพื่อแยกส่วนเซอร์วิส (Microservices) ให้เป็นสัดส่วน ป้องกันการตีกันของเวอร์ชันซอฟต์แวร์
* **โครงสร้างไดเรกทอรี:** `/opt/hotel-ecs/`
* **ระบบจัดการฐานข้อมูล:** SQLite (`hotel.db`) เก็บไว้ในโฟลเดอร์ `/opt/hotel-ecs/data` 
* **ตัวกลางเชื่อมต่อ (Connector):** เชื่อมต่อไปยัง IP ตู้สาขา `192.168.1.91` พอร์ต `23` (TCP/Telnet)

## 4. 🌐 การเปิดให้บริการสู่โลกภายนอก (Public Hosting via Tunnel)
เพื่อหลีกเลี่ยงการทำ Forward Port ที่เราเตอร์ซึ่งไม่ปลอดภัยและยุ่งยาก โครงการนี้จึงเลือกใช้ **Cloudflare Zero Trust (Cloudflare Tunnel)** เป็นอุโมงค์เชื่อมต่อแบบเข้ารหัส

1. **Cloudflare Tunnel:** ตัวเซิร์ฟเวอร์ Pi จะรันคอนเทนเนอร์ `cloudflared` เพื่อเจาะอุโมงค์ไปหา Cloudflare โดยใช้ Token เฉพาะตัว 
   * *สถานะล่าสุด:* กำหนดเส้นทาง (Public Hostname) ไปที่ `hotel.nithep.com` สำเร็จ
2. **DNS Management (Squarespace):** เนื่องจากโดเมน `nithep.com` จดทะเบียนและดูแล DNS โดย Squarespace เราจึงทำการสร้างเรคคอร์ดชี้เป้าหมายดังนี้:
   * **Type:** `CNAME`
   * **Name:** `hotel`
   * **Target:** `45e040a1-5b6f-4da6-bb65-7a16ef8a1762.cfargotunnel.com`
   * **การทำงาน:** เมื่อแขกพิมพ์ `https://hotel.nithep.com` -> Squarespace จะบอกให้วิ่งไปหา Cloudflare Tunnel -> Cloudflare จะส่งข้อมูลทะลุอุโมงค์กลับมาแสดงผลเว็บใน Raspberry Pi ทันที

> [!CAUTION]
> **ข้อมูลสำคัญสำหรับการจัดการ DNS และ Tunnel**
> - **ระบบจดโดเมน:** Squarespace (`admin@nithep.com`) *ต้องใช้รหัสจากแอป Authenticator ในมือถือเพื่อเข้าถึง (2FA)*
> - **ระบบจัดการ Tunnel:** Cloudflare Zero Trust
> - **Cloudflare Token:** จัดเก็บแยกต่างหากในไฟล์ `/opt/hotel-ecs/.env` บนเครื่อง Raspberry Pi ห้ามนำ Token จริงขึ้นสู่ระบบเปิด (GitHub/Git) เด็ดขาด

## 🛠️ แนวทางการจัดการและบำรุงรักษาในอนาคต (Next Steps & Maintenance)

1. **การป้องกันปัญหาไฟดับ (SD Card Corruption):**
   * *แนะนำ:* ควรพิจารณาเปิดใช้งานฟีเจอร์ **OverlayFS** ใน `raspi-config` บน Pi 4 เพื่อทำให้ระบบไฟล์หลักเป็นแบบอ่านอย่างเดียว (Read-Only) ป้องกันไฟล์ OS เสียหายเมื่อไฟดับกระชาก 
   * ส่วนไฟล์ฐานข้อมูล SQLite จะถูกแยกให้เขียนลงโฟลเดอร์เฉพาะ (Mount External/Read-Write partition)

2. **ระบบการอัปเดตโค้ด (Deployment):**
   * เมื่อมีการแก้ไขหน้าเว็บ UI ให้ใช้คำสั่ง `npm run build` จากคอมพิวเตอร์ผู้พัฒนา แล้วนำโฟลเดอร์ `dist` ไปอัปเดตลงใน Raspberry Pi
   * คอนเทนเนอร์สามารถสั่งรีสตาร์ทได้ง่ายๆ ด้วยคำสั่ง `docker-compose down && docker-compose up -d` ในโฟลเดอร์ `/opt/hotel-ecs/`

3. **การแก้ไขระบบ Wi-Fi กรณีเปลี่ยนเราเตอร์ (Recovery Mode):**
   * หากโรงแรมเปลี่ยนเราเตอร์ Wi-Fi หน้างานทำให้ Pi หลุดการเชื่อมต่อ สามารถใช้สาย LAN เสียบตรงจากคอมพิวเตอร์เข้าบอร์ด Pi และ SSH เข้าไปพิมพ์คำสั่ง `sudo nmcli device wifi connect "ชื่อไวไฟใหม่" password "รหัสใหม่"` ได้เลย

4. **การบำรุงรักษา Domain & Tunnel:**
   * ตรวจสอบวันหมดอายุของโดเมนใน Squarespace เสมอ (ควรเปิด Auto-renew)
   * หากพบว่าเข้าหน้าเว็บไม่ได้ (ขึ้น Error จาก Cloudflare) ให้ตรวจสอบ 2 จุดคือ:
     1. เครื่อง Pi 4 ไฟดับหรือหลุดอินเทอร์เน็ตหรือไม่?
     2. สาย LAN ที่ต่อตู้สาขาหลุดหรือไม่? (ถ้าระบบโหลดเว็บขึ้นแตะเปิดไฟไม่ได้)
