# 🚀 เริ่มบทใหม่ (Phase 2): การเชื่อมต่อตู้สาขาฮาร์ดแวร์จริง (Real Hardware Integration)

**วันที่เริ่มต้น:** 2 กรกฎาคม 2026
**สถานะปัจจุบัน:** 🟢 ซอฟต์แวร์เสร็จสมบูรณ์ (Production Ready บน Pi Z2W)
**เป้าหมายหลักในเฟสนี้:** นำ Raspberry Pi Zero 2 W ไปเชื่อมต่อกับตู้สาขาโทรศัพท์ของโรงแรม (**Phonik ECS-103R V.5**) เพื่อสั่งเปิด-ปิดกระแสไฟฟ้า 220V ภายในห้องพักจริงๆ

## 📋 เช็คลิสต์ก่อนเริ่มงาน (Pre-requisites)

- [x] โค้ด Backend และ Frontend ทำงานบน Pi Z2W อย่างเสถียร
- [x] พิสูจน์แล้วว่าคำสั่ง (Protocol) สามารถแปลงและยิงผ่านเครือข่ายได้จริง
- [x] สคริปต์ทดสอบพร้อมใช้ (`probe-pbx.js`, `test-relay.js`)

- [ ] สายสัญญาณ (Serial RS-232 หรือ LAN) สำหรับต่อจากตู้ PBX เข้า Pi Z2W
- [ ] ความเข้าใจเรื่องพอร์ต (Port) บนตู้สาขาที่ทางช่างเดินสายเตรียมไว้ให้

| รายการ | ค่า |
|--------|-----|

| **โหมดเชื่อมต่อ** | TCP/LAN |

## 🔌 การเตรียมตัวเชื่อมต่อฮาร์ดแวร์

เราต้องเลือกโหมดการเชื่อมต่อกับตู้จริงให้ตรงกับสายสัญญาณที่มีอยู่ ซึ่งระบบของเรารองรับไว้พร้อมแล้ว 2 รูปแบบ:

### ตัวเลือกที่ 1: เชื่อมต่อผ่าน Serial Port (RS-232)
*เหมาะสำหรับตู้สาขารุ่นเก่า หรือการต่อสายตรงเข้าบอร์ด*
หากใช้สายแปลง USB to RS-232 เสียบเข้าที่ Pi Z2W เราจะต้องแก้ค่าในไฟล์ `/home/admin/hotel-ecs-checkin/backend/.env` ดังนี้:
```env
PBX_MODE=serial
PBX_SERIAL=/dev/ttyUSB0
PBX_BAUD=9600
```

### ตัวเลือกที่ 2: เชื่อมต่อผ่าน LAN (TCP/IP) ✅ เลือกใช้แล้ว
*เหมาะสำหรับตู้สาขาที่มีพอร์ตแลน หรือมีตัวแปลง RS-232 to Ethernet*
ถ้าตู้สาขาเสียบสายแลนอยู่ เราจะใช้โหมด TCP แบบเดียวกับที่เราทดสอบ Simulator แต่เปลี่ยน IP ไปหาตู้จริง:
```env
PBX_MODE=tcp
PBX_HOST=192.168.1.91
PBX_PORT=23
```

## 🖥️ Phase 2A: ทดสอบบนเครื่อง Windows ก่อน (ก่อนลง Pi)

### แนวคิด
เนื่องจากโค้ดของเราเป็น Node.js (Cross-Platform) และตู้ PBX เชื่อมต่อผ่าน LAN ดังนั้นการรันโค้ดบน Windows จะให้ผลลัพธ์เดียวกับการรันบน Pi Z2W ทุกประการ ข้อดีคือพัฒนาเร็ว แก้โค้ดง่าย ไม่ต้อง deploy ลงบอร์ด

### เตรียมเครื่อง Windows
1. เครื่อง Windows ต้องอยู่ในวง LAN เดียวกับตู้ PBX (`192.168.1.x`)
2. ติดตั้ง Node.js (v18+)
3. ติดตั้ง AnyDesk (ถ้าต้องการรีโมทเข้าทดสอบ)

### ขั้นตอนทดสอบ 4 สเตป

#### สเตป 1: ทดสอบการเชื่อมต่อ (ปลอดภัย 100%)
```bash
cd pbx-connector
node probe-pbx.js 192.168.1.91 23 101
```
สคริปต์จะทดสอบ 3 ขั้นตอน:
- TCP Connect → เปิด Socket ได้ไหม?
- VERS Query → อ่าน Firmware Version (read-only, ไม่กระทบรีเลย์)
- ROOM Query → อ่านสถานะห้อง 101 (read-only)

#### สเตป 2: ทดสอบยิง ON/OFF ห้องจริง ⚡
```bash
# เปิดไฟห้อง 101 (จ่ายไฟ 220V!)
node test-relay.js 192.168.1.91 23 101 on

# ตรวจสอบสถานะ
node test-relay.js 192.168.1.91 23 101 status

# ปิดไฟห้อง 101 (ตัดไฟ)
node test-relay.js 192.168.1.91 23 101 off
```
> ⚠️ **คำเตือน**: คำสั่ง `on` และ `off` จะจ่าย/ตัดกระแสไฟ 220V จริง! ควรมีคนอยู่ที่ห้องทดสอบ

#### สเตป 3: รัน Backend + Frontend จริง
```bash
cd backend
npm start
```
Backend จะ:
- อ่าน `.env` → เชื่อมต่อตู้ PBX ผ่าน TCP (`192.168.1.91:23`)
- แสดง `[PBX] ✅ Connected in tcp mode`
- เริ่ม Heartbeat ทุก 30 วินาที
- เปิด HTTP Server ที่ Port 3000

#### สเตป 4: เปิด Browser กด Check-in!
- จากเครื่อง Windows: `http://localhost:3000`
- จากเครื่องอื่นในวง LAN: `http://<IP เครื่อง Windows>:3000`
- กด Check-in ห้อง 101 → ไฟในห้องควรสว่างขึ้น!
- กด Check-out ห้อง 101 → ไฟในห้องควรดับ!

### การรีโมทผ่าน AnyDesk
ถ้าคุณอยู่นอกสถานที่ แต่มีเครื่อง Windows ที่ต่อ LAN กับตู้ PBX อยู่แล้ว:
1. ติดตั้ง AnyDesk บนเครื่อง Windows ที่หน้างาน
2. รีโมทเข้าผ่าน AnyDesk จากเครื่องที่บ้าน
3. เปิด PowerShell → รัน `probe-pbx.js` ก่อน
4. พอ probe ผ่าน → รัน Backend → เปิด Browser กด Check-in ได้เลย!

## 🎯 Phase 2B: ย้ายลง Pi Z2W (เมื่อทดสอบบน Windows สำเร็จ)

1. **ต่อสายฮาร์ดแวร์**: เสียบสาย LAN จากตู้ PBX เข้าสู่ Pi Z2W (ผ่าน Switch/Router เดียวกัน)
2. **แก้ไข .env**: แก้ไขไฟล์ Configuration ให้ตรงกับ IP ที่ Pi เห็นตู้
3. **รีสตาร์ทระบบ**: พิมพ์ `pm2 restart hotel-backend` เพื่อโหลดค่าใหม่
4. **ตรวจสอบสถานะ (Ping)**: ดูผ่าน `pm2 logs` ว่าระบบแจ้งเตือน `[PBX] ✅ Connected` และได้รับ `💓 Heartbeat OK` จากตู้จริงหรือไม่
5. **ทดสอบยิงไฟฟ้า**: กด Check-in ห้องแรกผ่านเบราว์เซอร์ แล้วเดินไปดูไฟ 220V ในห้องพักว่าสว่างขึ้นหรือไม่!
